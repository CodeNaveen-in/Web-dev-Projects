    • Basic terms

    Web Browser

    Your personal assistant that allows you to browse through the internet and fetch the responses to your queries. Some most popular one are Google Chrome, Brave, Mozilla etc

    Web Server

    It is the both hardware and software responsible for storage and retrieval of the website's data. It acts as the central system that holds and responses to the requests by browser

    URL

    Uniform Resource Locator is the address that is used to access resources on the web

    It can be divided into 4 parts:

  • Protocol : https://
  • Domain: pizzayum.com
  • Path : /menu (Optional Part)
  • Query : ?id=123 (Optional part)

    • DNS

    Domain Name System is the internet's phonebook which directs browser to the server by changing the URl into IP address (numerical form that points to a server)

    Web Protocols

    Rules that allow all data transfer possible.

  • HTTP : Rules that allow transfer of data from browser to the server.
  • HTTPS: It is HTTP but more secure
  • TCP/IP: The base of HTTP that ensures data reassembly and delivery

    • Database

    It is the storage of all the information and connected with Web Server that does fetch, insert, update, delete depending on browser query and permissions.

    The database interactions can be done in either SQL or with NoSQL type database.

    Rendering

    Displaying the webpages is called rendering basic structure is HTML, styling is CSS and dynamic part is by Javascript.

    Browser renders in a Document Object Model where it stores a representation of the page in memory

    Web Security

    Security over the web is important for your and data safety

  • Encryption : SSL encrypts your data so that your details are not revealed
  • Authentication : The process of identifying your identity and authorisation is checking your permission based on which your request response is displayed.
  • Cookies : They are the small pieces of data that allow yoru browser to help remember minute details about your actions on the site and session management helps web server remember you.
  • Cross-site Scripting : Theya re the security threats that allow attackers to inject malicious script into your webpages and steal your details.

    • Important Terms

    API

    Application Programming Interface is the middleman in between wo software systems which allows them to talk to each other and exchange data in a structured way.

    JSON

    Javascript Object Notation is the universal langauge of data which is lightweight and human readable

    XML

  • It is used to store and trasnport data
  • It has user defiend tags
  • Does not display content visually
  • Tags are used to describe data
  • Very flexible in it's operation
    • XML is for systems while HTML is for browsers

    CMS

    Content Managment Systems are systems that allow companies to easily create, edit, manage and publish content without needing to code everytime

    It saves time, money, easily updates and is compatible with 3rd party integrations

    Wordpress and Payload are most popular examples

    Version Control

    It is the system that helps track changes to a project's codebase overtime.

    It allows multiple people to work on it simultaneously without conflicts and with all history of previous changes and ability to go back to them as well.

    Features of Version Control

    1. Team Collaboration
    2. History Tracking
    3. Experimentation
    4. Automatic Backup
    5. Work Transparency
